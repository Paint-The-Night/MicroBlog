Development Pipeline

This current development pipeline only exists to make it easier for me to publish easier. There is nothing in it's design that it can't just be uploaded straight to a web server and viewed like that. Saying that here is how I currently deploy

Local Editing (Visual Studio Code) -> Push to GitHub (GitHub Desktop) -> Deploy to S3 (AWS Code Pipeline) <- CDN Web Hosting (AWS CloudFront)

Web Hosting

This website is hosted in an Amazon S3 Bucket. S3 buckets do not allow for HTTPs connections. SO I've had to use the AWS CloudFront Platform as a Content Delivery Network (CDN)

Version Control

git is the undisputed standard for version control so it's been used in this blog. While it's not that important for collaborative work it let myself or anyone interested peel back the curtain to see any historical changes right from the begging of the project.

Text Editing

At the time of creations Visual Studio Code is the text editor I'm using to create this blog. Mostly for spell check and ease-of-use

Language

HTML is only real language you can for writing a website so it's been used. I would have preferred to write in markdown and I know there are tools for allowing that to happen - but thought it wasn't nessicary due to the design principles of this blog.
